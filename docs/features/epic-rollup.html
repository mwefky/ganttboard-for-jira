Epic & parent rollup

An epic in Jira is only as honest as the work underneath it. GanttBoard keeps the two in sync automatically: every parent with children draws a slim summary bar that’s derived from its children, so it can never quietly drift out of date.

  • Spans the real work — from the earliest child’s start to the latest child’s end.
  • Duration-weighted % complete — a three-week story counts for more than a one-day chore, so the number stays trustworthy.
  • Bottom-up through every level — sub-task → story → epic all roll up correctly.
  • Read-only by design — summary bars reflect their children and never overwrite the real epic dates in Jira. Drag the work underneath and the parent reshapes itself.

The result is a timeline you can actually report off: a short progress fill on a bar that stretches past today is your early warning that an epic is behind, long before anyone opens the individual stories. Pair it with critical path to see which child is driving the date, and read the full walkthrough on rolling up epic dates & progress in Jira.
